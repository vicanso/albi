module:middlewares/entry(processName, appUrlPrefixopt) → {function}

Source:
  • middlewares/entry.js, line 24

HTTP请求入口的中间件处理，包括：

  1. 对所有url做去前缀处理（nginx之类的反向代理基本url前缀做转发，未对url做处理）
  2. 如果X-User-Token未设置，设置为unknown
  3. 设置响应头的Via字段（从当前请求头中取Via再添加应用名字）
  4. 全局设置响应头Cache-Control:no-cache, max-age=0，避免请求未设置缓存属性，可以覆盖
  5. connectingTotal + 1 ，在请求处理完时， connectingTotal - 1
  6. 添加Timing对象至ctx.state.timing字段
  7. 在请求处理完成时，根据Timing的记录生成Server-Timing
Parameters:
Name Type Attributes Default Description
processName String

应用名字
appUrlPrefix String <optional>
 ''

应用的前缀URL，如果设置该参数，所有请求都做删除请前缀部分

返回中间件处理函数

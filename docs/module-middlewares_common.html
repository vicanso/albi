(static) noCache() → {function}

Source:
  • middlewares/common.js, line 51
See:
  • GitHub

用于校验请求是否设置了no-cache的相关属性，主要为了便于在varnish中更高效的判断该请求是不可缓存的 如果非GET或HEAD请求，不做任何校验，默认符合。 否则判断Header中的Cache-Control是否设置为no-cache或者querystring中是否有设置cache-control=no-cache。 注：如果使用Header的配置方式，会导致请求头中if-none-match等缓存相关的字段缺失，尽量使用querystring的方式

(static) noQuery() → {function}

Source:
  • middlewares/common.js, line 19

对于url中的querystring检验，如果有querystring则校验不通过，返回出错。 主要是一些接口是不需要querystring的，如果增加了，对接口处理没影响， 但是会导致varnish缓存的增多，特别有可能前端调用接口时把时间戳加上，完全利用不到varnish
